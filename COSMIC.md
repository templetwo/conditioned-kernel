## Cosmic + Helix (use when they help)

Prefer **`~/bin/cosmic-cli`** (or `export PATH="$HOME/bin:$PATH"`).  
`/usr/local/bin/cosmic` is a **stale launcher** pointing at a deleted Desktop path.

```bash
export PATH="$HOME/bin:$PATH"

cosmic-cli doctor
cosmic-cli helix status
cosmic-cli helix boot "conditioned kernel"
cosmic-cli helix recall "substrate gain OR arrival packet"
cosmic-cli helix state
cosmic-cli helix record 'insight text'
cosmic-cli helix thread 'unresolved question'

# Mission loop (boots Helix, sets goal, compass on SHELL/CODE)
cosmic-cli do 'M1: harden repair for goal_not_referenced'
cosmic-cli do --review 'careful change'
cosmic-cli review
cosmic-cli sessions
```

### When to reach for them

| Situation | Tool |
|---|---|
| Resume context / prior decisions | `helix boot` / `helix recall` |
| Session goal for this build window | set via `cosmic-cli do '…'` (auto) or helix_bridge |
| Durable insight from a green run | `helix record` |
| Unresolved research question | `helix thread` |
| Multi-step careful edit | `cosmic-cli do --review` |
| Independent second look at diffs | `cosmic-cli review` |
| Risky shell (rm, force-push, etc.) | Helix compass via Cosmic (WITNESS/PAUSE) |

### When not to

- Unit tests and `ck smoke` — run directly; faster, no model tax.
- Pure file edits you already understand — edit, then record if it matters.
- Anything that needs the broken `/usr/local/bin/cosmic` path — use `~/bin`.
